GitHub学生认证经验记录

发表于 2022-10-27 更新于 2023-01-28

动机

一开始是为了写 Swin-Unset 的代码（还是想要一套自己维护的、使用特别舒服的跑模型的代码）就想到使用 GitHub Copilot。那这个东西很久都不免费了，自然要求 GitHub 整一个学生认证才 OK。

第一次申请认证

  1. 进入 GitHub Education点击顶部的student，点击Student Program 进入Student页面。

  2. 之后点击页面上方的Get benefits for students，弹出要填写的第一张表

    这里要注意的是，网页上方的获取位置信息的权限一定要开“How do you plan to use GitHub?”这个问题还得好好填。

  3. 填完以后就是要填第二张表了，主要是要你交张证明。我交了张一网通办页面的截图，好像不行，提示我图像里找不到日期。

第二次申请认证

这一次我先注册了学校的邮箱。名字想了很久还是直接用了我名字的拼音。把学校的邮箱加入了我GitHub账号信息里去了证明也直接进了研究生管理系统的自助打印->在读证明，但是机器还是识别不了我的Date。所以我使用了翻译软件把中文的证明翻成了英文的才成功。不过，它一直提示这个东西：

You are significantly more likely to be verified if you have completed your GitHub user profile with your full name and a short bio.

但是我各种改个人信息都没用，所以第二次尝试以失败告终。

成功的尝试

在参考github学生认证一直被拒绝怎么办？ - 知乎 (zhihu.com)、记github学生认证_愧怍12的博客-CSDN博客_github学生认证 后，对申请流程做了些许调整：

  1. 证明使用学信网的证明而不是学校的证明。在学信网在线证明处使用Edge浏览器自带的翻译把简体中文翻译成英文。
  2. 在下方这个地方标注自己Github的用户名。不要那么老实地写中文拼音，内容对不上会不通过。
  3. 在提交完所有信息后，直接跳出一个界面和我说申请结果会在12天之内告诉我。这个时候返回GitHub Education主页申请就已经是成功的了！

Takeaways

  1. 网页上方的“获取位置信息”和“获取摄像头”的权限一定要开
  2. 认证时不要魔法上网
  3. 证书图片交学信网的在线验证报告的 Edge 浏览器翻译截图再P上自己的 GitHub 用户名，类型选others。
