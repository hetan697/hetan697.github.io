Windows Longhorn Build 4074

发表于 2022-10-04 更新于 2022-10-05

Longhorn 是 Microsoft 研发的用于接替 Windows XP 的操作系统代号，也就是 Windows Vista 的内部测试版。其中，Build 4074 可能是整个测试过程中唯一完全拥有完整、可用的 Aero 特效的系统，国庆期间在寝室试了试装了装这款系统的虚拟机玩，当然，没装成功。中间的一些步骤用这篇博客记录下来。

Longhorn 的改进与亮点

WinFS

借用百度百科的介绍：

WinFS 是一种新的文件系统格式。事实上，WinFS服务可以看作是在NTFS的基础上增加了一个数据库层，这个数据库层以即将出现的SQL Server的“Yukon”版为基础。对于WinFS来说，文件除了我们熟悉的属性，诸如文件名称、大小、日期外，还将通过诸如作者名、图像大小之类的元数据建立索引。系统底层的目录结构仍将存在，但用户使用的将是一个相似文件构成的库（Library）。每个库由一组通过查询WinFS数据库获得的文件构成。如果WinFS确实能将查找文件的依据改变为它是什么，而不是它在哪里的话，无疑这将是基于NTFS文件系统的一个突破。

这个概念是相当超前的。我们直至现在每次保存文件都要想清楚文件的文件名和存储路径。但是，首先，它真的有那么重要吗？如果我们只需要知道我们文件的内容就能索引并打开文件不是更好？再者，众所周知就像编程一样，最难做的事情其实是写注释和起变量名函数名，这种痛苦在一个普通的计算机使用者每次需要和文件打交道的时候居然也要承担，。

WinFS 在 NTFS 上添加了一个数据库作为耦合，用于查找单个文件的元数据，比如文档里面的具体内容等信息，其底层仍然是 NTFS 的树型文件结构。

所以这个想法可以说是相当超前了。

多想一点

我一贯是一个想得比较多的人。

也许在某一条时间线上，Longhorn 的负责人和微软上级并没有不合，也许这款系统最终带着华丽的界面成功发布，也许 WinFS 和最新最前沿的 web 技术已经带领我们改变了一切……
Longhorn 要真在 2004 年发布了，那就属于是外星科技了

想做的事情太多了，AI、Lisp、华丽的下一代系统（元宇宙？）……算了算了。
本来如果没有这一波疫情，我还能

说起来 Vista 各方面也都很好看，如果任务栏能做成和 Windows 7 一样就更好了

曾经的微软从来没有画过大饼
至少测试版系统还是搞出来了的

微软的审美和眼光都是在线且长远的：Win8 的卡牌式磁贴至今有人怀念，WP 的大标题栏正是现在 iOS 具有的，两者都以能将用户的注意力集中在核心内容上著名。

也正是 Longhorn 的暴死，微软才从 Vista 之后就一直只在系统上做一些小修小补的工作。Windows 7 就是个阉割版的 Vista；Windows 8 只是在 7 的基础上添加了一些适合平板的东西；Windows 8.1 还把改得最大的开始菜单改了回去；win 10 只是在 8.1 上换了套皮；win 11 又在 Win 10 上换了套皮。

现在 Win 11 的 C4D 做的是有那种感觉了（虽然只是 C4D但是好歹上道了），那么，微软什么时候把 NT 7 做出来呢？
